01 — Contexte

Un secteur réglementé sans outil numérique dédié

En France, l'amiante est interdit depuis 1997 mais demeure présent dans des millions de bâtiments construits avant cette date. La réglementation impose des diagnostics obligatoires avant toute vente, démolition ou travaux — réalisés exclusivement par des professionnels certifiés (SS3/SS4, Qualibat 1552, COFRAC BREv26/BREv28).

Problème côté clients

Trouver un professionnel certifié est difficile. Il n'existe aucune plateforme centralisée permettant de comparer les offres, vérifier les accréditations et suivre l'avancement d'un dossier amiante.

Problème côté professionnels

Les entreprises du secteur manquent de visibilité numérique et d'outils adaptés pour gérer leurs demandes de devis entrants et valoriser leurs certifications.

02 — Solution

Une marketplace verticale spécialisée

Désamianteurs.fr centralise l'offre et la demande sur une plateforme unique, sécurisée et transparente. Inspirée du modèle Travaux.com, elle est entièrement dédiée au secteur amiante avec des fonctionnalités métier spécifiques : vérification SIRET automatique, contrôle des certifications et algorithme de matching par abonnement.

Formulaire devis multi-étapes Saisie guidée du besoin (prestation, surface, délai, budget), géolocalisation Google Maps et création de compte en 3 étapes.
Annuaire des partenaires Recherche filtrée par type de prestation, zone géographique, certification et note moyenne.
Vérification SIRET automatique Edge Function Supabase proxiant l'API gouvernementale avec cache intégré et gestion des SIRET non diffusibles.
Algorithme de matching Classement des partenaires basé sur le tier d'abonnement (Élite → Freemium) — invisible pour les clients.
Onboarding PRO en 5 étapes Saisie SIRET, déclaration des certifications, zones d'intervention, types de prestations et choix d'abonnement.
Backoffice administrateur Validation des comptes PRO, modération des avis, supervision des KPI de la plateforme.
Authentification Google OAuth Connexion sociale avec détection et redirection automatique selon le rôle (client / pro / admin).
Email automation Notifications transactionnelles via n8n + Resend, hébergé sur VPS OVH (Roubaix, France).

07 — Défis techniques & apprentissages

Points clés de l'implémentation

RLS + DEFAULT auth.uid() Supabase / PostgreSQL
Les politiques INSERT évaluent WITH CHECK avant l'application des valeurs DEFAULT. Solution : utiliser WITH CHECK (TRUE) TO authenticated pour laisser la colonne user_id se renseigner automatiquement via DEFAULT auth.uid() sans la passer explicitement depuis WeWeb.
Préservation d'état entre étapes WeWeb — formulaire multi-étapes
WeWeb détruit le DOM lors du Conditional Rendering, effaçant les valeurs saisies. Solution : binding display (if(currentStep === 1, "flex", "none")) pour masquer sans démonter, préservant l'état de tous les champs entre les étapes.
Migration d'enum PostgreSQL Supabase — gestion des types
Impossible de supprimer une valeur d'enum directement en PostgreSQL. Pattern migratoire utilisé : création d'un nouveau type → migration de la colonne avec clause USING cast → DROP de l'ancien type → RENAME du nouveau. Les vues et politiques RLS doivent être supprimées et recréées dans la même migration.
Edge Function SIRET Supabase — API gouvernementale
Proxy sécurisé vers recherche-entreprises.api.gouv.fr avec gestion des cas edge : SIRET non diffusibles (retour 403 normalisé), entreprises cessées, et cache dans la table siret_cache pour limiter les appels API. Appelée depuis WeWeb via POST avec header Authorization: Bearer.
Syntaxe formule WeWeb WeWeb — spécificités no-code
Syntaxe propriétaire distincte du JavaScript standard : = pour la comparaison (pas ===), filterByKey(collection, "champ", valeur) pour les lookups, getByIndex(array, 0) pour l'accès aux éléments, if() pour les conditions. Pas de switch, de .replace() natif ni de chaînes de méthodes.
